# 毎日こんだてレビュー

人間中心設計の授業でユーザーレビューを行うための、スマホ向け料理献立アプリのプロトタイプです。

## 目的

忙しい大学生が、予算・調理時間・健康条件に合わせて献立を作り、買い物から調理、食後フィードバックまで進められるかを検証するための改善前プロトタイプです。

## 特徴

- 条件を入れなくても始められるホームと献立カレンダー
- 1食・1日・3日・1週間ごとの献立と、料理名つき買い物リスト
- 体調、調理時間、片付けの負担、栄養、味の系統を使った候補順位付け
- 欠品時の代替食材ランキングと、条件を維持した献立の再提案
- レシート読み取り風画面から購入済み・冷蔵庫データへの反映
- 外部API・ログイン・バックエンドなし
- 固定データによる献立、買い物リスト、代替食材、レシピ表示（和風・中華風・韓国風・アジア風・インド風を含む）
- スマホ表示を想定したWebアプリ

## 提案ロジック（レビュー用）

献立は、入力された項目だけを次の順で評価します。

1. アレルギーと苦手食材の除外
2. 調理時間
3. 予算
4. 洗い物・簡単さ・冷蔵庫の食材・その日の体調
5. たんぱく質・野菜・低脂質・バランス
6. 和風・中華風・韓国風・アジア風・インド風・洋風の希望

代替食材は、料理での置き換えやすさ、価格差、100gあたりのたんぱく質とエネルギー、調理時間への影響を点数化して上位5件を表示します。どの条件を維持しやすいかも候補ごとに表示します。

これはユーザーレビュー用の固定データによる評価であり、栄養指導やアレルギーの安全を保証するものではありません。
